# Loblaw Bio - Immune Cell Population Analysis

Analysis pipeline for Bob Loblaw's miraclib clinical trial data. Loads
cell-count.csv into SQLite, computes per-sample population frequencies,
compares responders vs non-responders in melanoma PBMC samples, and
summarizes the baseline subset. Includes a Streamlit dashboard.

## Quick start (GitHub Codespaces)

```bash
make setup       # install dependencies
make pipeline    # build db + run parts 2-4
make dashboard   # launch streamlit on port 8501
```

`make pipeline` runs, in order:

1. `load_data.py` — creates `cell_counts.db`
2. `scripts/analysis_part2.py` — frequency table
3. `scripts/analysis_part3.py` — stats + boxplots
4. `scripts/analysis_part4.py` — baseline subset summary

## Database schema

Normalized relational design in SQLite:

| Table | Purpose |
|---|---|
| `projects` | One row per trial project |
| `subjects` | Patient demographics (condition, age, sex) |
| `samples` | Sample metadata (treatment, response, timepoint, type) |
| `cell_populations` | Lookup for the 5 immune cell types |
| `cell_counts` | Fact table — one row per sample/population count |

**Why this shape:** Cell counts are stored in long format (sample + population + count) rather than five wide columns. Adding a new population is just new rows, not a schema migration. Subject-level fields live in `subjects` so they aren't duplicated across every sample. At scale (hundreds of projects, thousands of samples) this maps cleanly to Postgres — narrow fact table, indexed joins on small dimension tables, and easy aggregation for different analytics (longitudinal, cross-project, subset filtering).

## Part 2 — frequency table

For each sample, total cells = sum of all five populations. Relative
frequency = `count / total × 100`.

Output: `outputs/cell_frequencies.csv` with columns `sample`, `total_count`, `population`, `count`, `percentage`.

## Part 3 — responder vs non-responder

Filters to melanoma patients on miraclib with PBMC samples. Compares
relative frequencies between responders (`response = yes`) and
non-responders (`response = no`) using a two-sided Mann-Whitney U test
(non-parametric; group sizes and distributions differ).

Outputs:

- `outputs/responder_vs_nonresponder_stats.csv` — p-values per population
- `outputs/responder_comparison_data.csv` — underlying filtered data
- `outputs/plots/boxplot_*.png` — one boxplot per population

**Result:** Only **cd4_t_cell** shows a significant difference (p ≈ 0.013).

## Dashboard

Three tabs mirroring Parts 2–4: frequency table, responder analysis
with boxplots, and baseline breakdown. Reads from `cell_counts.db`
directly.

**Local:** `make dashboard` → http://localhost:8501

**GitHub repo:** https://github.com/jain-atishay/loblaw-bio-immune-analysis

**Deploy online:** Sign in at [share.streamlit.io](https://share.streamlit.io), connect the repo above, and set the app path to `dashboard/app.py`. Update the README with your Streamlit URL once deployed.

## Key findings (for Yah)

Among melanoma PBMC patients on miraclib, **cd4_t_cell** relative
frequency differs significantly between responders and non-responders
(Mann-Whitney U, p = 0.013). The other four populations do not reach
significance at α = 0.05.
